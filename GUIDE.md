# Raindrop.io Export CSV Documentation

## Overview

This export contains two CSV files that represent a complete hierarchical bookmark structure from Raindrop.io:

1. **`collections.csv`** - Metadata about all collections (folders)
2. **`all_raindrops_with_paths.csv`** - All bookmarks with their collection context

---

## Hierarchy Structure

```
Group (sidebar category, e.g., "Work", "Inspiration")
└── Collection (root level)
    └── Collection (nested)
        └── Collection (deeply nested)
            └── Bookmark (raindrop)
```

**Example full_path breakdown:**

```
Inspiration / Inspiration / Aleksandar / Competitors
    │              │          │          │
    │              │          │          └── Leaf collection (title)
    │              │          └── Parent collection
    │              └── Root collection (same name as group here)
    └── Group name
```

**Important distinctions:**

| Entity | Description | Has ID | Can Contain |
|--------|-------------|--------|-------------|
| **Group** | UI-only sidebar category label | No | Collections (by reference) |
| **Collection** | Actual folder | Yes | Bookmarks + Sub-collections |
| **Bookmark** | Individual saved URL | Yes | Nothing |

## File 1: `collections.csv`

Contains all collection/folder metadata. Use this to build navigation, filters, or folder trees.

### Columns

| Column | Type | Description | Example |
|--------|------|-------------|---------|
| `id` | Integer | Unique Raindrop collection ID | `63097098` |
| `title` | String | Collection name | `Scout Aleksandar` |
| `parentId` | Integer/Empty | ID of parent collection. Empty = root-level | `36074927` |
| `collectionPath` | String | Collection hierarchy path (without group) | `Inspiration / Aleksandar / Competitors` |
| `fullPath` | String | **Full path with group prefix** | `Inspiration / Inspiration / Aleksandar / Competitors` |
| `rootCollection` | String | Top-level ancestor collection name | `Inspiration` |
| `group` | String | Sidebar group this collection belongs to | `Inspiration` |

### Example Data

```csv
id,title,parentId,collectionPath,fullPath,rootCollection,group
63097098,mindquest,,mindquest,Work / mindquest,mindquest,Work
63169224,Scout Aleksandar,,Scout Aleksandar,Work / Scout Aleksandar,Scout Aleksandar,Work
50130604,All,,All,Discovery / All,All,Discovery
36202607,Competitors,36074927,Inspiration / Aleksandar / Competitors,Inspiration / Inspiration / Aleksandar / Competitors,Inspiration,Inspiration
55253163,Design Systems,,Design Systems,Discovery / Design Systems,Design Systems,Discovery
```

### Path Columns Explained

- **`collectionPath`**: Shows the collection hierarchy only (e.g., `Inspiration / Aleksandar / Competitors`)
- **`fullPath`**: Prepends the group name to show complete hierarchy (e.g., `Inspiration / Inspiration / Aleksandar / Competitors`)

Use `fullPath` when you need the complete folder structure including the sidebar group as the top level.

### Key Relationships

- `parentId` links to another collection's `id` for nested collections
- `parentId` is empty for root-level collections
- `group` is an organizational category (NOT a collection itself)
- Multiple collections can belong to the same group

## Common Query Patterns

### Filter bookmarks by group

```sql
SELECT * FROM bookmarks WHERE group = 'Work';
```

### Filter bookmarks by collection

```sql
SELECT * FROM bookmarks WHERE collection_id = 63097098;
```

### Get all bookmarks in a hierarchy (including nested)

```sql
SELECT * FROM bookmarks WHERE full_path LIKE 'Inspiration / Inspiration / Aleksandar%';
```

### Build breadcrumb navigation

Split `full_path` by " / " delimiter:

```javascript
const breadcrumbs = bookmark.full_path.split(' / ');
// ['Inspiration', 'Inspiration', 'Aleksandar', 'Competitors']
// First element is the group, second is the root collection
```

### Build folder tree from collections.csv

```javascript
// Create parent-child relationships
const collections = parseCSV('collections.csv');
const tree = {};

collections.forEach(col => {
  if (!col.parentId) {
    // Root level
    tree[col.id] = { ...col, children: [] };
  }
});

collections.forEach(col => {
  if (col.parentId && tree[col.parentId]) {
    tree[col.parentId].children.push(col);
  }
});
```

### Group bookmarks by group

```javascript
const grouped = bookmarks.reduce((acc, bookmark) => {
  const group = bookmark.group || 'Ungrouped';
  if (!acc[group]) acc[group] = [];
  acc[group].push(bookmark);
  return acc;
}, {});
```

## Notes

- **Deduplication**: Bookmarks are deduplicated by ID in the export
- **Empty values**: Some fields may be empty (notes, tags, cover, etc.)
- **Path separator**: Always ` / ` (space-slash-space)
- **DateTime format**: ISO 8601 (e.g., `2025-11-11T17:08:21.462Z`)
- **Boolean format**: `true` or `false` (lowercase strings)
